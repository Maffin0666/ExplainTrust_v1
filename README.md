# ExplainTrust — нечёткая экспертная система оценки когнитивного доверия пользователя к ответам языковых моделей

[![Python](https://img.shields.io/badge/python-3.10+-blue.svg)](https://python.org)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)
[![GitHub stars](https://img.shields.io/github/stars/Maffin0666/ExplainTrust_v1?style=social)](https://github.com/Maffin0666/ExplainTrust_v1)

**ExplainTrust** — это легковесная, интерпретируемая система, которая анализирует диалог между человеком и языковой моделью (LLM) и вычисляет **Индекс когнитивного доверия (Cognitive Trust Index, CTI)**.  
В основе лежит нечёткая логика Мамдани, а все признаки извлекаются с помощью простых лингвистических методов, без использования тяжёлых нейросетей.  
Проект выполнен в рамках лабораторной работы по дисциплине «Интеллектуальные информационные системы» и оформлен как полноценный программный продукт с графическим интерфейсом.

## 🚀 Возможности

- **Оценка одиночного ответа**: введите пару «запрос LLM — реакция пользователя» и получите CTI.
- **Анализ целого диалога**: загрузите лог переписки, система разобьёт его на пары и покажет динамику доверия.
- **Анализ активированных правил**: для любой пары можно увидеть, какие нечёткие правила сработали и с какой степенью.
- **Radar‑диаграмма признаков**: профиль выбранной пары в полярных координатах.
- **Тональность**: упрощённый словарный анализ позитивной/негативной окраски сообщений.
- **3D‑поверхность CTI**: исследование зависимости CTI от структурной целостности (SI) и маркеров сомнения (DM) при фиксированных остальных параметрах.
- **Психологический портрет пользователя**: текстовое описание типового поведения на основе усреднённых признаков.
- **Тёмная/светлая тема**: комфортная работа в любое время суток.
- **Экспорт отчётов**: сохранение результатов в текстовый файл.
- **Встроенные демо-диалоги**: несколько готовых примеров для быстрого старта.

## 🧠 Как это работает

Система анализирует два источника информации — **внешний локус** (качество ответа LLM) и **внутренний локус** (реакция пользователя).

### Признаки внешнего локуса (ответ LLM)

| Признак | Обозначение | Описание |
|---------|-------------|----------|
| Структурная целостность | SI | Наличие абзацев, вводных слов, списков. |
| Маркеры связности | CM | Доля дискурсивных слов: «поэтому», «однако», «во-первых». |
| Лексическое разнообразие | LD | Скользящее Type‑Token Ratio (MATTR) на основе стемминга. |

### Признаки внутреннего локуса (реакция пользователя)

| Признак | Обозначение | Описание |
|---------|-------------|----------|
| Маркеры сомнения | DM | Доля слов из словаря сомнений («возможно», «не уверен», «кажется»). |
| Переспрос | FQ | Наличие уточняющего вопроса (1 или 0). |
| Длина ответа | RL | Количество слов, нормированное на 50. |

### Нечёткий вывод

Признаки подаются на вход нечёткой системе с 12 продукционными правилами (алгоритм Мамдани).  
Каждое правило описывает эвристику, например:

> **Если** SI = высокая, CM = высокая, LD = высокое, DM = отсутствуют, FQ = нет, RL = средний **То** CTI = очень высокое

Выходной показатель CTI лежит в диапазоне **0…100**:

- 0–15 — очень низкое доверие
- 15–35 — низкое
- 35–55 — среднее
- 55–75 — высокое
- 75–100 — очень высокое

Все термы заданы треугольными и трапециевидными функциями принадлежности, агрегирование — максимумная композиция, дефаззификация — метод центра тяжести.

## 💻 Установка и запуск

### Требования

- Python 3.10 или новее
- pip
- Рекомендуется использовать виртуальное окружение

### Инструкция для PyCharm / терминала

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/Maffin0666/ExplainTrust_v1.git
   cd ExplainTrust
   ```

2. **Создайте и активируйте виртуальное окружение (опционально, но рекомендуется)**
   ```bash
   python -m venv .venv
   .venv\Scripts\activate   # Windows
   source .venv/bin/activate # Linux/Mac
   ```

3. **Установите зависимости**
   ```bash
   pip install -r requirements.txt
   ```
  Если файла requirements.txt нет, выполните:
  ```bash
  pip install nltk scikit-fuzzy numpy matplotlib networkx setuptools==68.0.0
  ```

4. **Скачайте данные NLTK (только при первом запуске)**
   ```bash
   python -c "import nltk; nltk.download('punkt'); nltk.download('punkt_tab')"
   ```

5. **Запустите графический интерфейс**
   ```bash
   python run_gui.py
   ```
   или для консольного теста:
      ```bash
   python run_example.py
   ```

## 🖥 Графический интерфейс

Приложение построено на базе `tkinter` и `matplotlib`. Все элементы имеют современный плоский дизайн (стиль `clam`), поддерживается переключение между светлой и **тёмной** темами.

### Вкладки

| Вкладка | Назначение |
|---------|------------|
| **Одиночный ответ** | Оценка одной пары «LLM – пользователь» |
| **Диалог** | Пакетный анализ целого диалога с динамикой CTI |
| **Анализ пар** | Детальная расшифровка активаций правил, radar‑диаграмма |
| **Тональность** | Зависимость CTI от тональности сообщений (scatter) |
| **3D‑анализ** | Визуализация поверхности CTI(SI, DM) |
| **Психопортрет** | Формирование словесного портрета пользователя |

Каждая вкладка снабжена **кнопкой справки** (знак `?`), поясняющей назначение элементов.

## 📊 Примеры использования

### Консольный запуск

```python
from explaintrust import ExplainTrust

engine = ExplainTrust()
result = engine.evaluate({
    "llm_response": "Согласно анализу, проблемы с индексами...",
    "user_reply": "Понял, спасибо!"
})
print(result['CTI'])          # например, 87.5
print(result['trust_level'])  # "очень высокое"
```

### Анализ диалога

```python
messages = [
    {"role": "llm", "content": "Основная причина — неоптимальные запросы."},
    {"role": "user", "content": "Понял, спасибо."},
    {"role": "llm", "content": "Также проверьте кэш."},
    {"role": "user", "content": "Хорошо, я проверю."}
]
turns = engine.evaluate_dialog_turns(messages)
for turn in turns:
    print(turn['CTI'], turn['trust_level'])
```

### Загрузка диалога из JSON
Файл должен содержать массив сообщений:
```json
[
  {"role": "llm", "content": "Текст ответа модели..."},
  {"role": "user", "content": "Реакция пользователя..."},
  ...
]
```
Этот файл можно открыть через GUI (кнопка «Загрузить JSON»).


## 📈 Визуализация и аналитика

Система предоставляет богатые возможности для исследования когнитивного доверия:

- **Динамика CTI** — график с цветовыми зонами уровней доверия.
- **Функции принадлежности** выходной переменной CTI с отмеченным результатом.
- **Scatter‑график** зависимости CTI от тональности сообщений LLM и пользователя.
- **3D‑поверхность** CTI как функции SI и DM при фиксированных CM, LD, FQ, RL.
- **Radar‑диаграмма** значений всех шести признаков для конкретной пары.
- **Психопортрет** — текстовый вывод, объединяющий усреднённые характеристики.

Все графики интерактивны (можно масштабировать, сохранять) благодаря `matplotlib`.
